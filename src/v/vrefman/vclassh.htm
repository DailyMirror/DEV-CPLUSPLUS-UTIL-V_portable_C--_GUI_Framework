V Class Hierarchy

The following figure is shows the internal organization of the V\ class hierarchy. Note that boxes with a double line edge denote classes that have object instances, while boxes with single line edges are abstract classes used to build subclasses. Most of the time, you won't care about these abstract classes.

Also note that the classes derived from vCmd represent the classes used to implement command objects. Normally, you won't need to used these classes directly. Instances of these objects are generated by V from your CommandObject declarations when you call AddDialogCmds.
